01

La démarche

L’objectif était simple : concevoir un portfolio à mon image, sans template ni framework superflu. Chaque détail — typographie, espacements, animations, couleurs — a été pensé et ajusté manuellement.

Le choix du vanilla JS, CSS et HTML est volontaire : moins de dépendances, c’est plus de contrôle, de meilleures performances et une sécurité renforcée. Le site est ainsi rapide, léger et entièrement maîtrisé.

« Simplicité et contrôle restent souvent les meilleures optimisations. »

02

Choix de design

  • [ Typo ]

    DM Serif Display

    Police serif élégante pour les titres — forte personnalité, lisible et distinctive sans être agressive.

  • [ Couleur ]

    Fond sombre + vert lime

    #0a0a0a comme base, #c8f0a0 comme accent — contraste maximal, lisibilité parfaite, identité mémorable.

  • [ Motion ]

    Animations CSS pures

    Scroll reveal via IntersectionObserver, hero avec stagger CSS, cursor personnalisé — aucun JS lourd requis.

  • [ Perf ]

    Zéro bundle, zéro overhead

    Aucun framework ni dépendance lourde — uniquement des fichiers statiques optimisés, servis directement via le CDN de Vercel pour un chargement rapide et constant.

03

Sécurité renforcée

Le site est sécurisé via des headers HTTP configurés sur Vercel. Ces protections permettent de limiter les risques d’attaques côté navigateur et de garantir des connexions sûres.

  • [ CSP ]

    Content-Security-Policy

    Restreint les sources de scripts, styles, fonts et images. Bloque toute injection XSS côté navigateur.

  • [ HSTS ]

    Strict-Transport-Security

    Force HTTPS pendant 2 ans, includeSubDomains + preload. Aucune connexion HTTP possible.

  • [ Frame ]

    X-Frame-Options: DENY

    Empêche le site d'être embarqué dans un iframe — protection contre le clickjacking.

  • [ Ref ]

    Referrer-Policy stricte

    strict-origin-when-cross-origin — ne divulgue l'URL complète qu'aux requêtes same-origin.

/* Scroll reveal — IntersectionObserver */
.reveal {
  opacity: 0;
