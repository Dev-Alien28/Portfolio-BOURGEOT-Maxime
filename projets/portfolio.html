01

La démarche

L'objectif était simple : construire un portfolio qui me ressemble, sans template, sans framework superflu. Chaque détail — typographie, espacements, animations, couleurs — a été décidé à la main.

Le choix du vanilla JS/CSS/HTML n'est pas une contrainte mais une décision : moins de dépendances = moins de surface d'attaque, moins de poids, plus de contrôle. Le site charge en une fraction de seconde.

« Le meilleur framework, c'est parfois pas de framework. »

02

Choix de design

  • [ Typo ]

    DM Serif Display

    Police serif élégante pour les titres — tranche nette, personnalité affirmée sans être agressive.

  • [ Couleur ]

    Fond sombre + vert lime

    #0a0a0a comme base, #c8f0a0 comme accent — contraste maximal, lisibilité parfaite, identité mémorable.

  • [ Motion ]

    Animations CSS pures

    Scroll reveal via IntersectionObserver, hero avec stagger CSS, cursor personnalisé — aucun JS lourd requis.

  • [ Perf ]

    Zéro bundle, zéro overhead

    Pas de build step, pas de node_modules. Juste des fichiers statiques servis directement depuis Vercel's CDN.

03

Sécurité renforcée

Le site est sécurisé via les headers HTTP de Vercel configurés dans vercel.json. La liste complète des protections mises en place :

  • [ CSP ]

    Content-Security-Policy

    Restreint les sources de scripts, styles, fonts et images. Bloque toute injection XSS côté navigateur.

  • [ HSTS ]

    Strict-Transport-Security

    Force HTTPS pendant 2 ans, includeSubDomains + preload. Aucune connexion HTTP possible.

  • [ Frame ]

    X-Frame-Options: DENY

    Empêche le site d'être embarqué dans un iframe — protection contre le clickjacking.

  • [ Ref ]

    Referrer-Policy stricte

    strict-origin-when-cross-origin — ne divulgue l'URL complète qu'aux requêtes same-origin.

/* Scroll reveal — IntersectionObserver */
.reveal {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity .5s ease, transform .5s ease;
  transition-delay: var(--reveal-delay, 0s);

  transform: none;
}
